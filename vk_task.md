# Задание 1

### Вступление 

Во вступелнии, как мне кажется, стоит сказать, что это должна быть поэтапная разработка рекомендательной системы, а не просто какой-то единичной модели. Учитывая что раньше персональных рекомендаций на главной странице не было, стоит правильно организовать сбор данных, запуск первой версии и продумать дальнейшие доработки/апгрейды.

### Основная цель

Сделать главную страницу более полезной для пользователя и увеличить вероятность того, что он найдет интересные ему товары.

### ML задача

Для пользователя нужно показать несколько товаров, которые с наибольшей вероятностью его заинтересуют.
Как мне кажется модель должна состоять из нескольких этапо выбор товара/сортировка и тд

#### Пример данных которые можно использовать: 
- история просмотров товаров
- клики по карточкам
- добавления в корзину
- покупки
- поисковые запросы
- интересующие категории

### Я бы начала с разработки простейшей версии для общего понимания работы и организцаии всего. Полсе этого придумывал бы более сложные решения.

Для начала создаем простую версию, которая позволит получить рабочий baseline и начать собирать данные.

### Дальше я бы строил систему в два этапа

- какой-то набор потенциально интересных товаров
- сортировка их под конкретного пользователя

## Для новых полльзователей
Если пользователь новый, можно показывать популярные товары или рекомендации по выбранным категориям.  
Если товар новый, можно опираться на его категорию, цену, бренд и описание.

## Важно так же запустить тестирование(оценку)

будем сомтреть на тесты и основыне метрики

# Задание 3

### Вступление
Так же разделим задачу на подзадачи. 
* Сначала нужно найти подходящих кандидатов, 
* а потом делать сортировку по вероятности того, что пользователь захочет на них подписаться.

### Основная цель
(в условии задания)

### ML задача

Строим список других пользователей, которых он с наибольшей вероятностью может зафолловить.

- набор подходящих кандидатов
- сортировка их по релевантности

#### Пример данных
- подписки пользователя
- общие подписки + подписчики
- общие подписчики
- контент который предпочитает пользователь

### Начала я бы тоже с более простго подхода

- рекомендовать друзей друзей
- пользователей с общими подписками

для начла это поможет понять что работате лучше
- потом уже можно доавбить предложения по контенту для пользователя

Дальше я бы строил систему из двух частей:
поиск кандидатов по подпискам + модель или алгоритм, который сортирует этих кандидатов по:
- количество общих связей
- похожие интересы(мейби)
- общие подписки
- активность и популярность аккаунта

 Для новых пользователей

- популярных пользователей
- аккаунты по выбранным интересам(в момент регистрации)
- людей из контактов(если ты и они давали свой доступ)

## Оценка качества

во время тестинга можем смотреть на:
- precision@K
- recall@K

После запуска смотреть на реальные продуктовые метрики:
- сколько людей перешли в профиль
- сколько подписались
- полезность реков
